Norme PHP ETNA

Cette norme a pour but d'uniformiser l'écriture des programmes au sein de l'ETNA.

Elle concerne :

  • La nomination des objets.
  • La présentation globale (paragraphes).
  • La présentation locale (lignes).
1) La nomination des objets

Les objets (variables, fonctions, classes) doivent avoir des noms les plus explicites ou mnémoniques et ce en ANGLAIS exclusivement.

Les abréviations sont TOLERÉES dans la mesure où elles permettent de réduire significativement la taille du nom sans en perdre le sens. De manière générale, une variable, fonction, classe... est écrite en miniscules.

  • Si son nom d'objet est composé, on sépare les différents mots qui le compose en mettant un underscore (_).

    • Exemple : $a_simple_var, function my_func().

  • Les noms de variables et de fonctions sont constitués uniquement de mininuscules, de chiffres et d'underscores.

    • Exemple :

    						
							<?php
							$aSimpleVar // Incorrect
							$a_simple_var // Correct
							$a_Simple_Var // Incorrect
							$ASimpleVar // Incorrect
							?>
  • Afin de distinguer les fonctions natives du language de celles que vous allez coder, toutes les fonctions que vous écrirez devront commencer par func_.

    • Exemple :

    						
							<?php
							function func_foo() // Correct
							function func_another_test() // Correct
							function foo() // Incorrect
							function func_4me() // Correct
							?>
2) Présentation globale
  • Les fichiers source (.php) doivent toujours commencer par le header standard de l'école. Ce header est créé sous Emacs à l'aide de la commande C-c C-h.
  • l'indentation se fait au moyen de la touche Tab sous Emacs, et un Alt-i pour la déclaration des fonctions.
  • On passe toujours à la ligne après un ; et une accolade ouvrante ou fermante ({, }).
  • L'accolade ouvrante se trouve sur la ligne suivant l'instruction qui provoque l'ouverture du bloc. Elle est alignée sur cette même instruction.
  • On indente a partir de la ligne qui suit l'accolade.

    • Exemple :

    						
							<?php
							function	func_my_test($a_simple_var)
							{
								echo "Hello World!";
							}
							?>
  • On met une variable par ligne et on ne saute jamais de ligne au sein d'une fonction.
  • Une ligne ne doit pas excéder 80 caractères.
  • On fera attention à faire des fonctions courtes et claires, de maximum 25 lignes.
  • Pas plus de 5 fonctions par fichier.
  • On ne fait jamais de commentaires dans une fonction, mais éventuellement au début de la fonction, et ce bien sûr en ANGLAIS.

    • Exemple :

    						
							<?php
							/*
							** A simple Hello World...
							*/
							function	func_hello_world($your_name)
							{
								$str = "Hello $your_name, how are you?";
								echo $str;
								return (true);
							}
							?>
  • La déclaration de paramètres se fera de la facon suivante :
    • 						
							<?php
							function	func_test($var1, $var2, $var3)
							{

							}
							?>
3) Présentation locale
  • Toujours un espace derrière la , et derrière le ; sauf en fin de ligne.
    • 						
							<?php
							for ($i = 0; $i < 10; $i++)
							{
								func_print_test(0, $i);
							}
							?>
  • Pas d'espace entre le nom d'une fonction et la (, par contre toujours un espace entre un mot clef PHP (avec argument) et la (.
    • 						
							<?php
							function	func_test($var1, $var2, $var3)
							{
								for ($i = $var1; $i < $var2; $i += $var3)
								{
									if ($i == 0)
									{
										print_r($i);
									}
									else
									{
										echo "$i\r\n";
									}
								}
								return ($var3);
							}
							?>
  • Ceci concerne for, while, etc... et aussi return.
  • Lorsque return prend un argument, ce dernier doit être entre parenthèses.
  • On alignera les déclarations avec des tabulations (sous Emacs M-i).
  • Affectations : Il doit y avoir un espace entre la variable de gauche et le signe d'affectation, de même pour la variable de droite.
  • Opérateurs unaires. Pas d'espaces.
    • 						
							<?php
							$test++;
							?>
  • Opérateurs bi/ternaires. Tous les opérateurs binaires et ternaires sont séparés des arguments par un espace de part et d'autre.
    • 						
							<?php
							if ($a % 10)
								return ($cp ? $cp : "DEFAULT STRING");
							?>
